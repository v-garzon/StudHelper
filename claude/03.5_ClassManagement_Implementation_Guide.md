# Class Management - Complete Implementation Guide

## 📋 Executive Summary

Complete class management system with:
- 3-step class creation wizard
- New permission model (14 permissions)
- File upload with processing queue
- Real-time document processing status view
- Blocked chat access until processing complete

---

## 🎯 Key Features

### ✅ Fully Functional
- Class creation with name & description
- File upload (PDF, DOCX, PPTX, TXT, MD)
- Permission presets (Reader, Student, Contributor, Manager)
- Max concurrent chats slider (3-15, default 5)
- Document processing queue
- Real-time processing status updates
- Navigation to processing view after creation

### ⚠️ Coming Soon (UI Ready)
- AI model selection (gpt-4o-mini/gpt-4o)
- YouTube URL processing (input field present)
- Sponsorship billing (toggle with warning tooltip)

## 🗄️ Database Schema Changes

### Migration: Update ClassMembership

```sql
-- Remove old fields
ALTER TABLE class_memberships DROP COLUMN is_manager;
ALTER TABLE class_memberships DROP COLUMN can_share_class;
ALTER TABLE class_memberships DROP COLUMN can_upload_documents;
ALTER TABLE class_memberships DROP COLUMN daily_token_limit;
ALTER TABLE class_memberships DROP COLUMN weekly_token_limit;
ALTER TABLE class_memberships DROP COLUMN monthly_token_limit;

-- Add new permission fields
ALTER TABLE class_memberships ADD COLUMN can_upload_files BOOLEAN DEFAULT TRUE;
ALTER TABLE class_memberships ADD COLUMN can_see_knowledge BOOLEAN DEFAULT TRUE;
ALTER TABLE class_memberships ADD COLUMN can_manage_knowledge BOOLEAN DEFAULT FALSE;
ALTER TABLE class_memberships ADD COLUMN can_see_permissions BOOLEAN DEFAULT FALSE;
ALTER TABLE class_memberships ADD COLUMN can_manage_permissions BOOLEAN DEFAULT FALSE;
ALTER TABLE class_memberships ADD COLUMN can_create_invite_codes BOOLEAN DEFAULT FALSE;
ALTER TABLE class_memberships ADD COLUMN can_revoke_members BOOLEAN DEFAULT FALSE;

-- Update token limits
ALTER TABLE class_memberships ADD COLUMN token_limit INTEGER DEFAULT 1000000;
ALTER TABLE class_memberships ADD COLUMN token_refresh_rate VARCHAR(20) DEFAULT 'daily';
```

### Migration: Add Class Settings

```sql
-- Add AI and permission defaults
ALTER TABLE classes ADD COLUMN ai_model VARCHAR(50) DEFAULT 'gpt-4o-mini';
ALTER TABLE classes ADD COLUMN default_permissions JSONB;
ALTER TABLE classes ADD COLUMN sponsorship_enabled BOOLEAN DEFAULT FALSE;
ALTER TABLE classes ADD COLUMN default_token_limit INTEGER;
ALTER TABLE classes ADD COLUMN default_token_refresh_rate VARCHAR(20);
```

### Migration: Add Document Processing Status

```sql
-- If not already present
ALTER TABLE documents ADD COLUMN processing_status VARCHAR(20) DEFAULT 'pending';
ALTER TABLE documents ADD COLUMN processing_error TEXT;
ALTER TABLE documents ADD COLUMN processing_started_at TIMESTAMP;
ALTER TABLE documents ADD COLUMN processing_completed_at TIMESTAMP;
```

### 1. Create Class (Enhanced)
```http
POST /api/v1/classes
Content-Type: application/json

{
  "name": "Physics 101",
  "description": "Introduction to Physics",
  "ai_model": "gpt-4o-mini",
  "default_permissions": {
    "can_read": true,
    "can_chat": true,
    // ... all permissions
  },
  "sponsorship_enabled": false,
  "default_token_limit": null,
  "default_token_refresh_rate": null
}

Response 201:
{
  "id": 123,
  "name": "Physics 101",
  "class_code": "ABC12345",
  "owner_id": 1,
  "created_at": "2025-01-15T10:00:00Z"
}
```

## 🔐 Permission Checks

### Chat Access
```javascript
// Before allowing chat creation
if (!allDocumentsCompleted) {
  throw "Documents are still being processed"
}

if (!userPermissions.can_chat) {
  throw "You don't have permission to chat"
}

if (userActiveChatCount >= userPermissions.max_concurrent_chats) {
  throw "Maximum concurrent chats reached"
}
```

### Knowledge Management
```javascript
// Before allowing document upload to class
if (!userPermissions.can_manage_knowledge) {
  throw "You don't have permission to manage class documents"
}
```
